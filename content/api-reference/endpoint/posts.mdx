### Advanced: pagination, sorting, and filtering

Use query parameters to paginate, sort, and filter the results. The example below returns the first page of posts in the "announcements" category that are tagged with either "product" or "engineering" and contain the term "modern", sorted by newest first.

<RequestExample>
  ```bash cURL (with params) curl
  "https://api.marblecms.com/v1/cm6ytuq9x0000i803v0isidst/posts?limit=10&page=1&order=desc&category=announcements&tags=product,engineering&query=modern"
  ```
</RequestExample>

### Query Parameters

<ParamField query="limit" type="integer | \"all\"" default="10">
  Maximum number of items to return. Use `all` to return all posts in a single response.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number to return. Applicable when `limit` is set to a number.
</ParamField>

<ParamField query="order" type="string" default="desc">
  Sort order by `publishedAt`. Accepted values: `asc`, `desc`.
</ParamField>

<ParamField query="category" type="string">
  Filter by category slug.
</ParamField>

<ParamField query="tags" type="string">
  Comma-separated list of tag slugs to filter by, e.g.
  `tags=product,engineering`.
</ParamField>

<ParamField query="query" type="string">
  Full-text search across `title` and `content`.
</ParamField>

<Note>
  Responses include a `pagination` object with `limit`, `currentPage`,
  `previousPage`, `nextPage`, `totalPages`, and `totalItems`. For advanced usage
  and error cases (like invalid page numbers), see the full guide on
  <a href="/content/api-reference/pagination">Pagination</a>.
</Note>
