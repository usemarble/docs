---
title: "Get a Single Post"
description: "Retrieves a single post by its slug or ID."
api: "GET /v1/posts/:identifier"
---

Fetch a specific post using either its unique slug or ID. Both identifiers work interchangeably.

<RequestExample>
```bash cURL
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://api.marblecms.com/v1/posts/introducing-marblecms"
```

```javascript JavaScript
const identifier = "introducing-marblecms"; // or use the post ID

const response = await fetch(
  `https://api.marblecms.com/v1/posts/${identifier}`,
  { headers: { Authorization: "Bearer YOUR_API_KEY" } }
);
const data = await response.json();
console.log(data.post);
```

</RequestExample>

## Authorizations

<ParamField header="Authorization" type="string" required>
  Your Marble API key. Include it in the header as: `Bearer YOUR_API_KEY`
</ParamField>

## Path Parameters

<ParamField path="identifier" type="string" required>
  The unique identifier for the post. Can be either: - **slug** - The
  URL-friendly post slug (e.g., `introducing-marblecms`) - **id** - The unique
  post ID (e.g., `clt92n38p000108l48zkj41an`)
</ParamField>

## Query Parameters

<ParamField query="format" type="string">
  Format of the `content` field in the response. | Value | Description | |
  Content converted to Markdown | ```bash curl -H "Authorization: Bearer
  YOUR_API_KEY" \
  "https://api.marblecms.com/v1/posts/introducing-marblecms?format=markdown" ```
</ParamField>

## Error Responses

<AccordionGroup>
<Accordion title="Post not found (404)">
```json
{
  "error": "Post not found",
  "message": "The requested post does not exist or is not published"
}
```
</Accordion>
</AccordionGroup>

<Tip>
  Use the slug for human-readable URLs in your frontend, and the ID for internal
  references where the slug might change.
</Tip>
