## Overview

Use this endpoint to retrieve all published blog posts from your Marble CMS workspace. The response includes post metadata, content, authors, categories, and tags with pagination support.

<RequestExample>
```bash cURL
curl "https://api.marblecms.com/v1/cm6ytuq9x0000i803v0isidst/posts"
```

```javascript Node.js
fetch("https://api.marblecms.com/v1/cm6ytuq9x0000i803v0isidst/posts")
  .then((res) => res.json())
  .then((json) => console.log(json))
  .catch((err) => console.error("error:" + err));
```

```python Python
import requests

response = requests.get("https://api.marblecms.com/v1/cm6ytuq9x0000i803v0isidst/posts")
print(response.json())
```
</RequestExample>

### Query Parameters

Use query parameters to paginate, sort, and filter the results.

<ParamField query="limit" type='integer | "all"' default="10">
  Maximum number of items to return. Use `all` to return all posts in a single
  response.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number to return. Applicable when `limit` is set to a number.
</ParamField>

<ParamField query="order" type="string" default="desc">
  Sort order by `publishedAt`. Accepted values: `asc`, `desc`.
</ParamField>

<ParamField query="category" type="string">
  Filter by category slug.
</ParamField>

<ParamField query="tags" type="string">
  Comma-separated list of tag slugs to filter by, e.g.
  `tags=product,engineering`.
</ParamField>

<ParamField query="query" type="string">
  Full-text search across `title` and `content`.
</ParamField>

<Note>
  Responses include a `pagination` object with `limit`, `currentPage`,
  `previousPage`, `nextPage`, `totalPages`, and `totalItems`. For advanced usage
  and error cases (like invalid page numbers), see the full guide on
  <a href="/api/pagination">Pagination</a>.
</Note>
